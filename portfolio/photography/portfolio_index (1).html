Git

Setting up:

Tutorial Video

Watch the video 'GIT Essential Training: The Basics' video. *Be sure you are logged into LinkedIn Learning before clicking the link. You can find the LinkedIn Learning site through myTVO.

Download GITKracken

Go to https://www.gitkraken.com/ to download the Git Client software.

Continue to sign up with your @tvo email address.

Login to GitLab TVO

Go to https://gitlab.tvo.org and signin using your TVO credentials.

Cloning the repo

Once successfully logged in, you should see the PROJECTS list. Choose the repo you wish to download (or "clone") to your htdocs folder by clicking its title.

Next, select the blue "Clone" dropdown button in the upper right hand corner of the screen and select 'Clone with HTTPS.'

Now that the link is copied to your clipboard, switch back over to GitKracken application and select FILE >> CLONE REPO and the following screen appears.

Click the blue BROWSE button, and select your 'htdocs' folder as the location where to clone the repo.

Paste the link copied from GitLab into the second box as the URL.

You should see a pop-up box letting you know the repo has been successfully cloned. Look in your htdocs folder on your machine to ensure there is a new folder there with the repo files.

*SPECIAL NOTE* Should you receive an error message, you may not have proper permissions to push/pull or clone a repo. In that case, see Derek Turney. In addition, you may see a header bar appear across the top of your GitKracken appplication asking you to enter your sign-on credentials.

Pushing and Pulling the repo

When you have made changes to your local copy repo (to HTML files, added images, renamed folders, etc) you will periodically need to update the repo for everyone else to see the changes you've made by "pushing" those changes via Git Kracken, which updates the repo directory on Gitlab.tvo.org.
